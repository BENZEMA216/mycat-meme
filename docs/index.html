案例 · The hardest one

挤眉弄眼，
但换成金渐层

这是整个项目最棘手的一个 case——源 meme 是一只橘猫的脸部特写，在做极其精细的挤眼鼓脸微表情。没有"场景"，没有"动作幅度"，只有一张脸和一个 vibes。

要求模型保留这个微表情，同时把猫换成一只完全不同品种的金色长毛小奶猫。

这一版跑了 7 次实验、3 次彻底失败，才找到 --description 这个关键开关。详细折腾记录在 v0.2.2 的 commit message 里。

$ mycat-meme replace-gif meme.gif my-cat.jpg \ -d "金色长毛小奶猫，蓬松长毛，圆脸幼态" \ -o out.gif

原理 · how it works

一条流水线，
三步到底。

  1. 拆 GIF，准备两个引用

    用 ffmpeg 把输入 GIF 提取首帧（静态构图参考），同时转成 h264 mp4（motion reference，给动作信号）。两者都按 dreamina multimodal2video 的硬约束做尺寸归一化。

  2. 丢给即梦 multimodal2video

    把 [你的猫照片, 首帧, motion mp4] 三个引用一次性送到 dreamina 的 Seedance 2.0 Fast 模型。用户的 --description 作为 prompt 锁定品种级特征。

  3. 下载 mp4，转回 GIF

    下载生成的 mp4（带 IncompleteRead 重试保护），用 ffmpeg 的 palette filter 做 2-pass 优化转回 GIF。全流程 3-5 分钟。

一个坦诚的局限：multimodal2video 有个怪脾气——只有参考图不够，prompt 里必须明确写出品种 / 毛色 / 毛长这类 keyword，否则模型会默认吐出一只"通用橘虎斑短毛"。所以 CLI 需要手动传 --description。v0.3 计划用 VLM 自动生成这一步。
安装 & 用法 · get started

五分钟，
装完就能跑。

需要先装即梦 CLI 并登录，然后用 pip 装 mycat-meme。ffmpeg 也是必需的（brew install ffmpeg）。

Step 01 · 装即梦 CLI 并登录

这是工具背后真正干活的引擎

# 见即梦 CLI 官方仓库的安装说明 dreamina login --headless dreamina user_credit # 验证登录 + 查看积分
Step 02 · 装 mycat-meme

一条 pip 命令

pip install mycat-meme
Step 03 · 跑一次看看

静态图 & GIF 两种用法

# 静态图 mycat-meme replace meme.png my-cat.jpg -o out.png # GIF （强烈建议加 -d 描述你家猫） mycat-meme replace-gif meme.gif my-cat.jpg \ -d "金色长毛小奶猫，蓬松长毛，圆脸幼态" \ -o out.gif
